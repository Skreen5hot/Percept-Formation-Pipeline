Percept-Formation-Pipeline

A percept-formation data pipeline: raw input refined into structural then semantic types, fallibly bound, then deterministically adjudicated.

Three points on the honesty ladder. Dirty: a driver column of mixed string/numeric ids — bound and met, yet flagged uncertain (usable, correct, shaky). Mislabeled: two vehicle columns and no driver — the adjudicator rejects the forced agent mapping and names the law. Clinical: a glucose table under a second law — the dictionary derives the unwitnessed performer, instrument, and (by inherence, from the value itself) the specimen. Disputed: a transaction that is honestly both a sale and a purchase — the adjudicator declines to choose, records both, and lights L4.

Structured-source front (Phase 2): a star schema (a fact table + dimensions + a human-ratified mapping) instead of raw bytes. The SSM front resolves each foreign key to its dimension row and binds it to a role, then hands the flat ActOfOrdering frame to the SAME Binder → OCE → FSDD core the raw samples use — same manifest, different door. Northwind: a 9-role star (date played in 3 roles) resolves clean. missing order_date: a constitutive role is empty → ICE (honest incompleteness, not a guess). orphan customer: a constitutive FK dangles → the whole frame is excluded and named.
